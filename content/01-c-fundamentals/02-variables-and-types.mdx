In many languages, you write `x = 42` and don't think about it. In C, you need to tell the compiler *exactly* what kind of data `x` holds, because that determines how many bytes of memory it occupies and how the CPU operates on it.

## Declaring Variables

In C, every variable must be declared with a **type** before you use it:

<Editor defaultCode={`#include <stdio.h>\n\nint main() {\n    int age = 25;\n    float height = 5.9;\n    char grade = 'A';\n    \n    printf("Age: %d\\n", age);\n    printf("Height: %.1f\\n", height);\n    printf("Grade: %c\\n", grade);\n    \n    return 0;\n}\n`} time="O(1)" space="O(1)" />

## The Basic Types

| Type | Size (typical) | Range | Use for |
|------|---------------|-------|---------|
| `char` | 1 byte | -128 to 127 | Single characters, small numbers |
| `int` | 4 bytes | ~-2 billion to ~2 billion | Whole numbers |
| `float` | 4 bytes | ~6 decimal digits precision | Decimal numbers |
| `double` | 8 bytes | ~15 decimal digits precision | Precise decimal numbers |
| `long` | 8 bytes | Very large whole numbers | When `int` isn't big enough |

<Callout type="warning">
Unlike dynamically-typed languages where variables can hold any value, C integers have fixed sizes. An `int` is typically 4 bytes (32 bits), meaning it can only represent values up to about 2.1 billion. This is a direct trade-off: fixed-size types are faster because the CPU has native 32-bit operations, but you have to think about limits.
</Callout>

## Type Modifiers

The basic types can be modified with `short`, `long`, and `unsigned` to change their size or range.

### Size Modifiers

| Type | Typical Size | Use for |
|------|-------------|--------|
| `short` (or `short int`) | 2 bytes | Saving memory when values are small |
| `int` | 4 bytes | General-purpose integers |
| `long` (or `long int`) | 8 bytes | Large values, timestamps |
| `long long` (or `long long int`) | 8 bytes | Guaranteed at least 64 bits |

### signed vs unsigned

By default, integer types are **signed** ‚Äî they can hold negative values. Adding `unsigned` drops the negative half and **doubles the positive range**:

- `int`: -2,147,483,648 to 2,147,483,647
- `unsigned int`: 0 to 4,294,967,295

Why use `unsigned`? Three reasons:
1. **Sizes and counts** are never negative ‚Äî `sizeof` returns `size_t`, which is unsigned
2. **Bit manipulation** is cleaner with unsigned types (no sign bit to worry about)
3. **Protocol/file formats** often define fields as unsigned

<Editor defaultCode={`#include <stdio.h>\n\nint main() {\n    printf("short:         %zu bytes\\n", sizeof(short));\n    printf("int:           %zu bytes\\n", sizeof(int));\n    printf("long:          %zu bytes\\n", sizeof(long));\n    printf("long long:     %zu bytes\\n", sizeof(long long));\n    printf("unsigned int:  %zu bytes\\n", sizeof(unsigned int));\n    \n    unsigned int positive_only = 4000000000U;\n    printf("\\nunsigned int value: %u\\n", positive_only);\n    \n    return 0;\n}\n`} time="O(1)" space="O(1)" />

<Callout type="info">
The exact sizes of `short`, `int`, `long` are **platform-dependent** ‚Äî the C standard only guarantees minimum sizes (e.g., `int` is at least 16 bits). This is why `<stdint.h>` provides fixed-width types like `int32_t` and `uint64_t`, which we'll cover later.
</Callout>

## Format Specifiers

`printf` uses **format specifiers** to know how to print each type:

| Specifier | Type | Example |
|-----------|------|---------|
| `%d` | int | `printf("%d", 42)` ‚Üí `42` |
| `%f` | float/double | `printf("%f", 3.14)` ‚Üí `3.140000` |
| `%.2f` | float (2 decimals) | `printf("%.2f", 3.14)` ‚Üí `3.14` |
| `%c` | char | `printf("%c", 'A')` ‚Üí `A` |
| `%s` | string (char array) | `printf("%s", "hello")` ‚Üí `hello` |
| `%ld` | long | `printf("%ld", 100000L)` ‚Üí `100000` |

## sizeof ‚Äî Checking Type Sizes

The `sizeof` operator tells you how many bytes a type or variable uses:

<Editor defaultCode={`#include <stdio.h>\n\nint main() {\n    printf("char:   %zu bytes\\n", sizeof(char));\n    printf("int:    %zu bytes\\n", sizeof(int));\n    printf("float:  %zu bytes\\n", sizeof(float));\n    printf("double: %zu bytes\\n", sizeof(double));\n    printf("long:   %zu bytes\\n", sizeof(long));\n    \n    return 0;\n}\n`} time="O(1)" space="O(1)" />

<Callout type="info">
`%zu` is the format specifier for `size_t`, which is what `sizeof` returns. It's an unsigned integer type guaranteed to be large enough to represent the size of any object in memory.
</Callout>

## The const Keyword

The `const` qualifier tells the compiler a variable's value must not change after initialization:

```c
const int MAX_STUDENTS = 100;
const double PI = 3.14159265358979;
```

If you try to modify a `const` variable, the compiler will reject your code. This matters for three reasons:

1. **Bug prevention** ‚Äî the compiler catches accidental modifications
2. **Documenting intent** ‚Äî readers know this value is fixed
3. **Optimization** ‚Äî the compiler can make assumptions about unchanging values

### const with Pointers

This is where `const` gets tricky. Read pointer declarations **right to left**:

```c
const int *p;      // p is a pointer to a const int ‚Äî can't change the value
int *const p;      // p is a const pointer to int ‚Äî can't change the pointer
const int *const p; // both the pointer and value are const
```

- `const int *p` ‚Äî you can reassign `p` to point elsewhere, but you can't modify `*p`
- `int *const p` ‚Äî you can modify `*p`, but you can't reassign `p` itself

You'll encounter `const int *` frequently when a function promises not to modify data you pass to it (like `strlen(const char *s)`).

<Callout type="warning">
Unlike `#define`, a `const` variable in C is **not a true compile-time constant**. In C89/C90, you cannot use `const int N = 10;` to declare an array like `int arr[N];` ‚Äî the compiler requires a constant expression. Use `#define N 10` or switch to C99's variable-length arrays. C++ treats `const` differently here.
</Callout>

## Integer Overflow

What happens when a number gets too big for its type?

<Editor defaultCode={`#include <stdio.h>\n#include <limits.h>\n\nint main() {\n    int max = INT_MAX;  // Largest possible int\n    printf("INT_MAX:     %d\\n", max);\n    printf("INT_MAX + 1: %d\\n", max + 1);  // Overflow!\n    \n    return 0;\n}\n`} time="O(1)" space="O(1)" />

<Callout type="warning">
Integer overflow in C is **undefined behavior** for signed integers. The compiler can do *anything*. In practice, it usually wraps around (2147483647 + 1 = -2147483648), but you must never rely on this. Later, we'll see how undefined behavior leads to real security vulnerabilities.
</Callout>
