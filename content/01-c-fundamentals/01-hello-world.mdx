Every programmer starts here. But unlike a tutorial that just tells you to type `printf` and move on, we're going to understand *exactly* what happens when you write and run a C program.

## Your First Program

Here's the classic:

<Editor defaultCode={`#include <stdio.h>\n\nint main() {\n    printf("Hello, world!\\n");\n    return 0;\n}\n`} time="O(1)" space="O(1)" />

Click **Run** and you should see `Hello, world!` in the output panel. Now let's break down every single line.

## What Each Line Does

### `#include <stdio.h>`

This is a **preprocessor directive**. Before the compiler even looks at your code, a program called the preprocessor runs first. `#include` literally copies the contents of a file called `stdio.h` into your source code.

`stdio.h` stands for **Standard Input/Output Header**. It declares functions like `printf` (print formatted output) and `scanf` (read formatted input).

<Callout type="info">
The preprocessor literally copy-pastes the entire header file into your source code before compilation begins. This is different from a module or import system — it's raw text insertion. We'll explore the preprocessor in much more detail later.
</Callout>

### `int main()`

This declares a **function** called `main` that returns an `int` (integer). Every C program must have exactly one `main` function — it's where execution begins.

The `int` return type matters: your program returns a number to the operating system. `0` means "everything went fine." Any other number means an error occurred.

<Callout type="info">
When you run a command in your terminal and then check `echo $?`, you're seeing the return value of that program's `main()` function. This is how scripts and tools detect whether a command succeeded or failed.
</Callout>

### `printf("Hello, world!\n")`

`printf` = **print formatted**. It writes text to **stdout** (standard output — usually your terminal).

The `\n` is a **newline character**. Without it, your terminal prompt would appear right after "Hello, world!" on the same line.

### `return 0;`

Returns the integer `0` to the operating system, signaling successful execution. After this line, your program exits.

## The Compilation Process

Unlike Python or JavaScript, C is a **compiled language**. Your source code goes through several stages before it becomes a program the computer can run:

1. **Preprocessing** — `#include` directives are resolved, macros are expanded
2. **Compilation** — C code is translated to assembly language
3. **Assembly** — Assembly is translated to machine code (object files)
4. **Linking** — Object files are combined with library code to produce an executable

On Linux, the standard compiler is **GCC** (GNU Compiler Collection):

```
gcc -o hello hello.c    # Compile hello.c into an executable called 'hello'
./hello                 # Run it
```

<Callout type="info">
When you hit "Run" on this website, your code is sent to a server running GCC inside a secure sandbox. The same compile-link-run cycle happens behind the scenes every time.
</Callout>

## Multiple printf Calls

You can call `printf` multiple times. Each call picks up where the last one left off:

<Editor defaultCode={`#include <stdio.h>\n\nint main() {\n    printf("Line 1\\n");\n    printf("Line 2\\n");\n    printf("Still line 2... ");\n    printf("because no newline above\\n");\n    return 0;\n}\n`} time="O(1)" space="O(1)" />

## Escape Sequences

The `\n` we used is called an **escape sequence**. C has several:

| Sequence | Meaning |
|----------|---------|
| `\n` | Newline |
| `\t` | Tab |
| `\\` | Backslash |
| `\"` | Double quote |
| `\0` | Null character |

<Editor defaultCode={`#include <stdio.h>\n\nint main() {\n    printf("Col1\\tCol2\\tCol3\\n");\n    printf("A\\tB\\tC\\n");\n    printf("She said \\"hello\\"\\n");\n    return 0;\n}\n`} time="O(1)" space="O(1)" />
