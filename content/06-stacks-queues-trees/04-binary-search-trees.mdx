A **binary search tree** (BST) is a binary tree where for every node: all values in the left subtree are smaller, and all values in the right subtree are larger. This single rule — the **BST property** — turns a tree into a search engine.

## Why BSTs Give You Binary Search

Think about how binary search works on a sorted array: you look at the middle element, and if your target is smaller, you discard the right half. Each comparison eliminates half the remaining elements.

A BST does exactly the same thing, but with pointers instead of indices. At the root, you compare your target to the root's value. If it's smaller, you go left — discarding the entire right subtree. If it's larger, you go right — discarding the entire left subtree. Each step eliminates roughly half the remaining nodes (when the tree is balanced).

The difference from a sorted array: you can *insert* and *delete* without shifting elements. A sorted array requires O(n) to insert (everything after the insertion point shifts right). A BST inserts in O(h) — just walk down to the right spot and attach a new leaf.

## Building a BST

Inserting into a BST is beautifully recursive. If the tree is empty, create a node. Otherwise, compare the new value to the current node and recurse left or right. The new value always lands as a leaf — it never needs to displace existing nodes.

<Editor defaultCode={`#include <stdio.h>
#include <stdlib.h>

typedef struct Node {
    int data;
    struct Node *left;
    struct Node *right;
} Node;

Node *create_node(int data) {
    Node *n = malloc(sizeof(Node));
    n->data = data;
    n->left = NULL;
    n->right = NULL;
    return n;
}

// Insert into BST, return root
Node *insert(Node *root, int data) {
    if (root == NULL) return create_node(data);

    if (data < root->data) {
        root->left = insert(root->left, data);
    } else if (data > root->data) {
        root->right = insert(root->right, data);
    }
    // if data == root->data, skip (no duplicates)
    return root;
}

void inorder(Node *node) {
    if (node == NULL) return;
    inorder(node->left);
    printf("%d ", node->data);
    inorder(node->right);
}

void free_tree(Node *n) {
    if (!n) return;
    free_tree(n->left);
    free_tree(n->right);
    free(n);
}

int main() {
    Node *root = NULL;
    int values[] = {50, 30, 70, 20, 40, 60, 80};

    for (int i = 0; i < 7; i++) {
        root = insert(root, values[i]);
    }

    //       50
    //      /  \\
    //    30    70
    //   / \\   / \\
    //  20 40 60 80

    printf("In-order (sorted): ");
    inorder(root);
    printf("\\n");

    free_tree(root);
    return 0;
}
`} time="O(h) per operation" space="O(n)" />

The in-order traversal prints `20 30 40 50 60 70 80` — sorted. This is guaranteed by the BST property: in-order visits left (smaller values), then root, then right (larger values). It's a natural consequence of how the tree is organized.

<Callout type="info">
**In-order traversal of a BST always produces sorted output.** This is the key insight — the BST property guarantees ordering. If your in-order traversal isn't sorted, your tree isn't a valid BST.
</Callout>

## Search: The Binary Search Analogy in Action

Searching a BST is where the "binary search" part of the name becomes clear. At each node, you make one comparison and eliminate an entire subtree — just like binary search eliminates half the array.

Let's trace searching for 40 in the tree above:
- Start at 50. Is 40 < 50? Yes → go left.
- At 30. Is 40 > 30? Yes → go right.
- At 40. Found it!

Three comparisons to find the value, skipping nodes 70, 20, 60, 80 entirely. Each comparison cuts the search space. In a balanced tree with n nodes, the height is approximately `log2(n)`, so search takes O(log n).

Now search for 25:
- Start at 50 → go left.
- At 30 → go left.
- At 20 → go right.
- NULL → not found.

When you reach NULL, the value doesn't exist in the tree. The search path is the path where the value *would* be inserted.

<Editor defaultCode={`#include <stdio.h>
#include <stdlib.h>

typedef struct Node {
    int data;
    struct Node *left;
    struct Node *right;
} Node;

Node *create_node(int d) {
    Node *n = malloc(sizeof(Node));
    n->data = d; n->left = NULL; n->right = NULL;
    return n;
}

Node *insert(Node *root, int d) {
    if (!root) return create_node(d);
    if (d < root->data) root->left = insert(root->left, d);
    else if (d > root->data) root->right = insert(root->right, d);
    return root;
}

Node *search(Node *root, int target) {
    if (root == NULL || root->data == target) {
        return root;
    }
    if (target < root->data) {
        return search(root->left, target);
    }
    return search(root->right, target);
}

void free_tree(Node *n) {
    if (!n) return;
    free_tree(n->left); free_tree(n->right); free(n);
}

int main() {
    Node *root = NULL;
    int vals[] = {50, 30, 70, 20, 40, 60, 80};
    for (int i = 0; i < 7; i++) root = insert(root, vals[i]);

    int targets[] = {40, 25, 70, 100};
    for (int i = 0; i < 4; i++) {
        Node *result = search(root, targets[i]);
        printf("Search %d: %s\\n", targets[i],
               result ? "found" : "not found");
    }

    free_tree(root);
    return 0;
}
`} time="O(h)" space="O(h) recursion" />

## Finding Min and Max

The smallest value is the leftmost node, the largest is the rightmost. This follows directly from the BST property: left is always smaller, so if you keep going left, you reach the smallest value. No comparisons needed — just follow the left pointers until you can't anymore.

<Editor defaultCode={`#include <stdio.h>
#include <stdlib.h>

typedef struct Node {
    int data;
    struct Node *left;
    struct Node *right;
} Node;

Node *create_node(int d) {
    Node *n = malloc(sizeof(Node));
    n->data = d; n->left = NULL; n->right = NULL;
    return n;
}
Node *insert(Node *r, int d) {
    if (!r) return create_node(d);
    if (d < r->data) r->left = insert(r->left, d);
    else if (d > r->data) r->right = insert(r->right, d);
    return r;
}

Node *find_min(Node *node) {
    while (node->left != NULL) {
        node = node->left;
    }
    return node;
}

Node *find_max(Node *node) {
    while (node->right != NULL) {
        node = node->right;
    }
    return node;
}

void free_tree(Node *n) {
    if (!n) return;
    free_tree(n->left); free_tree(n->right); free(n);
}

int main() {
    Node *root = NULL;
    int vals[] = {50, 30, 70, 20, 40, 60, 80};
    for (int i = 0; i < 7; i++) root = insert(root, vals[i]);

    printf("Min: %d\\n", find_min(root)->data);
    printf("Max: %d\\n", find_max(root)->data);

    free_tree(root);
    return 0;
}
`} time="O(h)" space="O(1)" />

## Deletion: The Hardest BST Operation

Deletion is tricky because removing a node must preserve the BST property. There are three cases, each progressively harder.

### Case 1: Deleting a Leaf (no children)

This is easy. The node has no children, so just remove it — set the parent's pointer to NULL.

```
Before:         After deleting 20:
      50              50
     /  \            /  \
   30    70        30    70
  / \              / \
 20  40              40
```

### Case 2: Deleting a Node with One Child

Replace the node with its only child. The child (and its entire subtree) slides up to take the deleted node's place. The BST property is preserved because the child's subtree was already on the correct side.

```
Before:          After deleting 30 (which has children 20, 40):
      50         Wait — 30 has TWO children. Let's pick a better example.
     /  \
   30    70      Before:         After deleting 70:
  / \     \            50              50
 20  40    80         /  \            /  \
                    30    70        30    80
                   / \     \       / \
                  20  40    80    20  40
```

Node 70 had one child (80). We remove 70 and 80 takes its place. Simple.

### Case 3: Deleting a Node with Two Children

This is the hard case. You can't just remove the node — it has two subtrees that need to stay connected. The solution: **replace the node's value with its in-order successor (the smallest value in its right subtree), then delete the successor.**

Why the in-order successor? Because it's the next-largest value in the tree. Putting it in the deleted node's position preserves the BST property: everything in the left subtree is still smaller (the successor is larger than all of them), and everything remaining in the right subtree is still larger (the successor was the smallest among them).

```
Before:              After deleting 50:
      50                   60
     /  \                 /  \
   30    70             30    70
  / \   / \            / \     \
 20 40 60  80         20 40    80

Step 1: Find in-order successor of 50 → it's 60 (smallest in right subtree)
Step 2: Copy 60's value into node 50's position
Step 3: Delete the original 60 from the right subtree (it's a leaf — Case 1)
```

The successor is always easy to delete — it has at most one child (a right child). Think about why: the successor is the *leftmost* node in the right subtree. If it had a left child, that child would be smaller — contradicting that the successor is the leftmost.

<Editor defaultCode={`#include <stdio.h>
#include <stdlib.h>

typedef struct Node {
    int data;
    struct Node *left;
    struct Node *right;
} Node;

Node *create_node(int d) {
    Node *n = malloc(sizeof(Node));
    n->data = d; n->left = NULL; n->right = NULL;
    return n;
}
Node *insert(Node *r, int d) {
    if (!r) return create_node(d);
    if (d < r->data) r->left = insert(r->left, d);
    else if (d > r->data) r->right = insert(r->right, d);
    return r;
}

Node *find_min(Node *n) {
    while (n->left) n = n->left;
    return n;
}

Node *delete_node(Node *root, int key) {
    if (root == NULL) return NULL;

    if (key < root->data) {
        root->left = delete_node(root->left, key);
    } else if (key > root->data) {
        root->right = delete_node(root->right, key);
    } else {
        // Found the node to delete
        if (root->left == NULL) {
            Node *temp = root->right;
            free(root);
            return temp;
        }
        if (root->right == NULL) {
            Node *temp = root->left;
            free(root);
            return temp;
        }
        // Two children: use in-order successor
        Node *successor = find_min(root->right);
        root->data = successor->data;
        root->right = delete_node(root->right, successor->data);
    }
    return root;
}

void inorder(Node *n) {
    if (!n) return;
    inorder(n->left);
    printf("%d ", n->data);
    inorder(n->right);
}

void free_tree(Node *n) {
    if (!n) return;
    free_tree(n->left); free_tree(n->right); free(n);
}

int main() {
    Node *root = NULL;
    int vals[] = {50, 30, 70, 20, 40, 60, 80};
    for (int i = 0; i < 7; i++) root = insert(root, vals[i]);

    printf("Before: ");
    inorder(root);
    printf("\\n");

    root = delete_node(root, 50);  // delete root (two children)
    printf("After deleting 50: ");
    inorder(root);
    printf("\\n");

    root = delete_node(root, 20);  // delete leaf
    printf("After deleting 20: ");
    inorder(root);
    printf("\\n");

    free_tree(root);
    return 0;
}
`} time="O(h)" space="O(h)" />

Notice how the code handles Cases 1 and 2 together: if `left == NULL`, return `right` (which may also be NULL — that's Case 1). If `right == NULL`, return `left`. This elegance comes from treating "no child" (NULL) and "one child" the same way.

## BST Complexity

| Operation | Average | Worst (degenerate) |
|-----------|---------|-------------------|
| Search | O(log n) | O(n) |
| Insert | O(log n) | O(n) |
| Delete | O(log n) | O(n) |

The "average" assumes the tree is roughly balanced — each level has about twice as many nodes as the level above. The height is then approximately `log2(n)`, giving logarithmic operations.

<Callout type="warning">
If you insert sorted data into a BST, it becomes a linked list (all nodes go right). This is the worst case — height equals n, and every operation becomes O(n). Imagine inserting 1, 2, 3, 4, 5: each value is larger than the previous, so every node goes right, forming a single chain. Balanced BSTs (AVL, Red-Black) fix this by rotating nodes to keep the tree balanced, but they're beyond our scope here.
</Callout>
