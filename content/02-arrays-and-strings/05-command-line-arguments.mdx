So far, our programs get input from `scanf` or `fgets` during execution. But you can also pass data to a program when you launch it — these are **command-line arguments**.

## argc and argv

The `main` function can take two parameters:

<Editor defaultCode={`#include <stdio.h>\n\nint main(int argc, char *argv[]) {\n    printf("Number of arguments: %d\\n", argc);\n    \n    for (int i = 0; i < argc; i++) {\n        printf("argv[%d] = %s\\n", i, argv[i]);\n    }\n    \n    return 0;\n}\n`} time="O(n)" space="O(1)" />

- **`argc`** (argument count) — the number of arguments, including the program name
- **`argv`** (argument vector) — an array of strings (char pointers), one per argument

When you run `./program hello world`, you get:
```
argc = 3
argv[0] = "./program"
argv[1] = "hello"
argv[2] = "world"
```

<Callout type="info">
`argv[0]` is always the program name (or the path used to invoke it). The actual user arguments start at `argv[1]`. So a program with "no arguments" still has `argc == 1`.
</Callout>

## Using Arguments

A common pattern is to check `argc` and use arguments:

<Editor defaultCode={`#include <stdio.h>\n#include <stdlib.h>\n\nint main(int argc, char *argv[]) {\n    if (argc != 3) {\n        printf("Usage: %s <num1> <num2>\\n", argv[0]);\n        return 1;\n    }\n    \n    // Convert string arguments to integers\n    int a = atoi(argv[1]);\n    int b = atoi(argv[2]);\n    \n    printf("%d + %d = %d\\n", a, b, a + b);\n    \n    return 0;\n}\n`} time="O(1)" space="O(1)" />

## Converting Arguments

Arguments are always strings. To use them as numbers, you need to convert:

| Function | Converts to | Header |
|----------|------------|--------|
| `atoi(str)` | `int` | `<stdlib.h>` |
| `atol(str)` | `long` | `<stdlib.h>` |
| `atof(str)` | `double` | `<stdlib.h>` |
| `strtol(str, NULL, 10)` | `long` (with error checking) | `<stdlib.h>` |

<Callout type="tip">
`atoi` silently returns 0 on invalid input. For production code, prefer `strtol` which provides error checking. For learning exercises, `atoi` is fine.
</Callout>

## How argv Works in Memory

`argv` is an array of pointers. Each pointer points to a null-terminated string:

```
argv:  [ptr0] [ptr1] [ptr2] [NULL]
         |      |      |
         v      v      v
       "./prog" "hello" "world"
```

The last element `argv[argc]` is guaranteed to be `NULL`.
