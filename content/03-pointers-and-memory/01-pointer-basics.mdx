Pointers are often called the hardest concept in C. But they're really just variables that hold memory addresses. Once you understand that, everything clicks.

## What Is a Pointer?

Every variable in your program lives at a specific **memory address**. A pointer is a variable that stores such an address.

<Editor defaultCode={`#include <stdio.h>\n\nint main() {\n    int x = 42;\n    int *ptr = &x;  // ptr holds the address of x\n    \n    printf("Value of x:      %d\\n", x);\n    printf("Address of x:    %p\\n", (void *)&x);\n    printf("Value of ptr:    %p\\n", (void *)ptr);   // Same address\n    printf("Value at ptr:    %d\\n", *ptr);           // 42 (dereference)\n    \n    return 0;\n}\n`} time="O(1)" space="O(1)" />

Two key operators:
- **`&x`** (address-of) — gives the memory address of variable `x`
- **`*ptr`** (dereference) — gives the value stored at the address `ptr` points to

## Declaring Pointers

A pointer variable is declared with `*` after the type:

```c
int *p;       // pointer to int
char *c;      // pointer to char
double *d;    // pointer to double
float *f;     // pointer to float
```

<Callout type="info">
The type matters. An `int *` knows it points to 4 bytes of data interpreted as an integer. A `char *` points to 1 byte interpreted as a character. The type tells the compiler how to interpret the memory and how far to move with pointer arithmetic.
</Callout>

## Modifying Through Pointers

Since a pointer gives you access to a variable's memory location, you can modify the original variable through the pointer:

<Editor defaultCode={`#include <stdio.h>\n\nint main() {\n    int a = 10;\n    int *p = &a;\n    \n    printf("Before: a = %d\\n", a);\n    \n    *p = 99;  // Modify a through the pointer\n    \n    printf("After:  a = %d\\n", a);  // 99!\n    \n    return 0;\n}\n`} time="O(1)" space="O(1)" />

This is how C simulates "pass by reference" — by passing the address of a variable to a function.

## NULL Pointers

A pointer that doesn't point to anything valid should be set to `NULL`:

<Editor defaultCode={`#include <stdio.h>\n\nint main() {\n    int *p = NULL;\n    \n    if (p == NULL) {\n        printf("Pointer is NULL\\n");\n    }\n    \n    // Dereferencing NULL causes a crash (segmentation fault)\n    // *p = 5;  // DON'T DO THIS\n    \n    // Always check before dereferencing\n    if (p != NULL) {\n        printf("Value: %d\\n", *p);\n    } else {\n        printf("Cannot dereference NULL\\n");\n    }\n    \n    return 0;\n}\n`} time="O(1)" space="O(1)" />

<Callout type="warning">
Dereferencing a NULL pointer causes a **segmentation fault** — your program crashes. Always initialize pointers to NULL and check before dereferencing.
</Callout>

## Pointer Size

All pointers are the same size on a given system, regardless of what type they point to:

<Editor defaultCode={`#include <stdio.h>\n\nint main() {\n    printf("int *:    %zu bytes\\n", sizeof(int *));\n    printf("char *:   %zu bytes\\n", sizeof(char *));\n    printf("double *: %zu bytes\\n", sizeof(double *));\n    printf("void *:   %zu bytes\\n", sizeof(void *));\n    \n    // On a 64-bit system, all pointers are 8 bytes\n    // On a 32-bit system, all pointers are 4 bytes\n    \n    return 0;\n}\n`} time="O(1)" space="O(1)" />
