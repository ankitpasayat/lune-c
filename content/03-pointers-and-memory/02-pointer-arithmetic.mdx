Pointers and arrays are deeply connected in C. Understanding pointer arithmetic unlocks the ability to work with data at the memory level.

## Arrays and Pointers

An array name acts as a pointer to its first element:

<Editor defaultCode={`#include <stdio.h>\n\nint main() {\n    int arr[] = {10, 20, 30, 40, 50};\n    int *p = arr;  // p points to arr[0] (no & needed)\n    \n    printf("arr[0] = %d\\n", *p);       // 10\n    printf("arr[1] = %d\\n", *(p + 1)); // 20\n    printf("arr[2] = %d\\n", *(p + 2)); // 30\n    \n    // These are equivalent:\n    printf("\\narr[3] = %d\\n", arr[3]);     // Bracket notation\n    printf("arr[3] = %d\\n", *(arr + 3));  // Pointer arithmetic\n    \n    return 0;\n}\n`} time="O(1)" space="O(1)" />

<Callout type="info">
In C, `arr[i]` is literally defined as `*(arr + i)`. The bracket notation is just syntactic sugar for pointer arithmetic. This is why array indices start at 0: `arr[0]` = `*(arr + 0)` = the element at the start.
</Callout>

## Incrementing Pointers

When you add 1 to a pointer, it moves forward by the **size of the pointed-to type**, not by 1 byte:

<Editor defaultCode={`#include <stdio.h>\n\nint main() {\n    int arr[] = {10, 20, 30, 40, 50};\n    int *p = arr;\n    \n    printf("p   points to %d (address %p)\\n", *p, (void *)p);\n    p++;  // Move to next int (forward 4 bytes)\n    printf("p+1 points to %d (address %p)\\n", *p, (void *)p);\n    p++;  // Move to next int (forward another 4 bytes)\n    printf("p+2 points to %d (address %p)\\n", *p, (void *)p);\n    \n    return 0;\n}\n`} time="O(1)" space="O(1)" />

This is why pointer types matter:
- `int *p; p++` moves forward 4 bytes (sizeof int)
- `char *p; p++` moves forward 1 byte (sizeof char)
- `double *p; p++` moves forward 8 bytes (sizeof double)

## Iterating with Pointers

You can traverse an array using a pointer instead of an index:

<Editor defaultCode={`#include <stdio.h>\n\nint main() {\n    int arr[] = {1, 2, 3, 4, 5};\n    int n = 5;\n    \n    // Index-based iteration\n    printf("Index:   ");\n    for (int i = 0; i < n; i++) {\n        printf("%d ", arr[i]);\n    }\n    printf("\\n");\n    \n    // Pointer-based iteration\n    printf("Pointer: ");\n    for (int *p = arr; p < arr + n; p++) {\n        printf("%d ", *p);\n    }\n    printf("\\n");\n    \n    return 0;\n}\n`} time="O(n)" space="O(1)" />

## Pointer Subtraction

Subtracting two pointers gives the number of elements between them:

<Editor defaultCode={`#include <stdio.h>\n\nint main() {\n    int arr[] = {10, 20, 30, 40, 50};\n    int *start = &arr[0];\n    int *end = &arr[4];\n    \n    printf("Elements between: %ld\\n", end - start);  // 4\n    \n    // Useful for finding an element's index\n    int *found = &arr[2];\n    printf("Index of 30: %ld\\n", found - arr);  // 2\n    \n    return 0;\n}\n`} time="O(1)" space="O(1)" />

<Callout type="warning">
You can only subtract pointers that point into the same array. Subtracting pointers to unrelated variables is undefined behavior.
</Callout>

## Pointer Comparison

Pointers into the same array can be compared:

```c
int arr[5];
int *p = &arr[1];
int *q = &arr[3];

if (p < q)   // true: p points to an earlier element
if (p == q)  // false: different positions
if (p != q)  // true
```

## Functions with Pointer Parameters

Passing an array to a function always passes a pointer:

<Editor defaultCode={`#include <stdio.h>\n\n// These three declarations are equivalent:\n// void print_array(int arr[], int n)\n// void print_array(int *arr, int n)\nvoid print_array(int *arr, int n) {\n    for (int i = 0; i < n; i++) {\n        printf("%d ", arr[i]);  // Can use [] with pointers too\n    }\n    printf("\\n");\n}\n\nvoid double_elements(int *arr, int n) {\n    for (int *p = arr; p < arr + n; p++) {\n        *p *= 2;  // Modifies the original array!\n    }\n}\n\nint main() {\n    int data[] = {1, 2, 3, 4, 5};\n    \n    print_array(data, 5);\n    double_elements(data, 5);\n    print_array(data, 5);  // All values doubled\n    \n    return 0;\n}\n`} time="O(n)" space="O(1)" />
