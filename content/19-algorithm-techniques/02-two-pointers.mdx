Consider this problem: given a **sorted** array and a target sum, find two numbers
that add up to the target.

The brute force is to check all pairs — two nested loops, O(n²). But the array is
sorted. That structure is a gift, and we're wasting it.

## The Insight

Put one finger on the smallest element (left end) and one on the largest (right end).
Add them. If the sum is too small, move the left finger right to increase it. If
the sum is too large, move the right finger left to decrease it. Each step eliminates
an entire row of possibilities.

This is the **two pointers** technique — two indices that cooperate to scan an array
faster than nested loops could.

## Opposite Ends: Two Sum (Sorted)

<Editor defaultCode={`#include <stdio.h>

void two_sum_sorted(int arr[], int n, int target) {
    int left = 0, right = n - 1;

    while (left < right) {
        int sum = arr[left] + arr[right];

        if (sum == target) {
            printf("Found: arr[%d]=%d + arr[%d]=%d = %d\\n",
                   left, arr[left], right, arr[right], target);
            return;
        } else if (sum < target) {
            left++;   // sum too small — need a bigger left value
        } else {
            right--;  // sum too big — need a smaller right value
        }
    }
    printf("No pair found for target %d\\n", target);
}

int main() {
    int arr[] = {1, 3, 5, 7, 9, 11, 15};
    int n = 7;

    two_sum_sorted(arr, n, 12);  // 3 + 9
    two_sum_sorted(arr, n, 16);  // 1 + 15
    two_sum_sorted(arr, n, 100); // not found
    return 0;
}
`} time="O(n)" space="O(1)" />

### Why Does This Work?

Think about it carefully. When `arr[left] + arr[right] < target`, could the answer
involve `arr[left]` paired with something *smaller* than `arr[right]`? No — because
we already tried the largest possible partner for `arr[left]` and it wasn't enough.
So `arr[left]` can't be part of any solution. We can safely eliminate it.

The same logic applies when the sum is too large: `arr[right]` can't be part of any
solution, so we eliminate it. Each step prunes a candidate. With at most `n` steps,
we've either found the answer or proven none exists.

<Callout type="info">
If the array isn't sorted, sort it first for O(n log n) + O(n) = O(n log n) total.
Still better than O(n²). Alternatively, use a hash table for O(n) — but that's a
different technique entirely.
</Callout>

## Opposite Ends: Palindrome Check

Two pointers are the natural tool for any problem that compares elements from both
ends of a sequence:

<Editor defaultCode={`#include <stdio.h>
#include <string.h>

int is_palindrome(char *s) {
    int left = 0;
    int right = strlen(s) - 1;

    while (left < right) {
        if (s[left] != s[right]) {
            return 0;  // Mismatch found — not a palindrome
        }
        left++;
        right--;
    }
    return 1;  // All characters matched
}

int main() {
    char *tests[] = {"racecar", "hello", "madam", "ab", "a"};
    int n = 5;

    for (int i = 0; i < n; i++) {
        printf("\\"%s\\" is %sa palindrome\\n",
               tests[i],
               is_palindrome(tests[i]) ? "" : "not ");
    }
    return 0;
}
`} time="O(n)" space="O(1)" />

No new arrays, no string reversal, no extra memory. Just two indices walking inward.

## Same Direction: The Fast/Slow Pattern

The other major variant: both pointers start at the beginning and move right, but at
different speeds. The slow pointer marks a "boundary" — everything before it satisfies
some condition. The fast pointer scans ahead looking for elements to place.

### Remove Duplicates from Sorted Array

<Editor defaultCode={`#include <stdio.h>

int remove_duplicates(int arr[], int n) {
    if (n == 0) return 0;

    int slow = 0;  // Everything up to slow is unique

    for (int fast = 1; fast < n; fast++) {
        if (arr[fast] != arr[slow]) {
            // New unique element found — place it after slow
            slow++;
            arr[slow] = arr[fast];
        }
        // If arr[fast] == arr[slow], fast moves but slow stays put
    }
    return slow + 1;  // New length
}

int main() {
    int arr[] = {1, 1, 2, 2, 2, 3, 4, 4, 5};
    int n = 9;

    printf("Before: ");
    for (int i = 0; i < n; i++) printf("%d ", arr[i]);
    printf("\\n");

    int new_len = remove_duplicates(arr, n);

    printf("After:  ");
    for (int i = 0; i < new_len; i++) printf("%d ", arr[i]);
    printf("  (length: %d)\\n", new_len);

    return 0;
}
`} time="O(n)" space="O(1)" />

The mental model: `slow` is the gatekeeper. `fast` explores ahead. When `fast` finds
something new, it presents it to `slow`, who accepts it and advances. When `fast` finds
a duplicate, `slow` ignores it and `fast` keeps going.

## Same Direction: Reverse Array In-Place

This is technically "opposite ends" in disguise — but it shows how the pattern
adapts:

<Editor defaultCode={`#include <stdio.h>

void reverse(int arr[], int n) {
    int left = 0, right = n - 1;
    while (left < right) {
        int tmp = arr[left];
        arr[left] = arr[right];
        arr[right] = tmp;
        left++;
        right--;
    }
}

int main() {
    int arr[] = {1, 2, 3, 4, 5, 6, 7};
    int n = 7;

    printf("Before: ");
    for (int i = 0; i < n; i++) printf("%d ", arr[i]);

    reverse(arr, n);

    printf("\\nAfter:  ");
    for (int i = 0; i < n; i++) printf("%d ", arr[i]);
    printf("\\n");

    return 0;
}
`} time="O(n)" space="O(1)" />

## When to Reach for Two Pointers

Ask these questions:
1. Is the input **sorted** or can you sort it? → Opposite-ends pattern likely works
2. Need to modify an array **in-place** with O(1) space? → Fast/slow pattern
3. Are you comparing elements from **both ends** (palindromes, matching brackets)? → Opposite ends
4. Are you **partitioning** an array (zeros to one side, negatives to the left)? → Fast/slow
