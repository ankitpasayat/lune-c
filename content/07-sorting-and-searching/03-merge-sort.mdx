Merge sort is the first **O(n log n)** algorithm we'll learn. It uses the divide-and-conquer strategy: split the problem in half, solve each half, then combine the results.

## The Idea

1. **Divide**: Split the array into two halves
2. **Conquer**: Recursively sort each half
3. **Combine**: Merge the two sorted halves

<Editor defaultCode={`#include <stdio.h>

void merge(int arr[], int left, int mid, int right) {
    int n1 = mid - left + 1;
    int n2 = right - mid;

    // Temporary arrays
    int L[n1], R[n2];

    for (int i = 0; i < n1; i++) L[i] = arr[left + i];
    for (int i = 0; i < n2; i++) R[i] = arr[mid + 1 + i];

    // Merge back
    int i = 0, j = 0, k = left;
    while (i < n1 && j < n2) {
        if (L[i] <= R[j]) {
            arr[k++] = L[i++];
        } else {
            arr[k++] = R[j++];
        }
    }

    // Copy remaining
    while (i < n1) arr[k++] = L[i++];
    while (j < n2) arr[k++] = R[j++];
}

void merge_sort(int arr[], int left, int right) {
    if (left >= right) return;  // base case: 0 or 1 element

    int mid = left + (right - left) / 2;
    merge_sort(arr, left, mid);       // sort left half
    merge_sort(arr, mid + 1, right);  // sort right half
    merge(arr, left, mid, right);     // merge
}

int main() {
    int arr[] = {38, 27, 43, 3, 9, 82, 10};
    int n = 7;

    merge_sort(arr, 0, n - 1);

    for (int i = 0; i < n; i++) {
        printf("%d ", arr[i]);
    }
    printf("\\n");

    return 0;
}
`} time="O(n log n)" space="O(n)" />

<Complexity time="O(n log n)" space="O(n)" />

## How The Merge Works

The `merge` function is the key. Given two sorted subarrays, it combines them into one sorted array:

```
Left:  [3, 27, 38, 43]
Right: [9, 10, 82]

Compare: 3 < 9  → take 3
Compare: 27 > 9 → take 9
Compare: 27 > 10 → take 10
Compare: 27 < 82 → take 27
Compare: 38 < 82 → take 38
Compare: 43 < 82 → take 43
Remaining: take 82

Result: [3, 9, 10, 27, 38, 43, 82]
```

Each merge takes O(n) time, and there are O(log n) levels of recursion, giving O(n log n) total.

## Trace Through the Recursion

<Editor defaultCode={`#include <stdio.h>

int depth = 0;

void print_indent() {
    for (int i = 0; i < depth; i++) printf("  ");
}

void merge(int arr[], int l, int m, int r) {
    int n1 = m - l + 1, n2 = r - m;
    int L[n1], R[n2];
    for (int i = 0; i < n1; i++) L[i] = arr[l + i];
    for (int i = 0; i < n2; i++) R[i] = arr[m + 1 + i];

    int i = 0, j = 0, k = l;
    while (i < n1 && j < n2)
        arr[k++] = (L[i] <= R[j]) ? L[i++] : R[j++];
    while (i < n1) arr[k++] = L[i++];
    while (j < n2) arr[k++] = R[j++];

    print_indent();
    printf("Merged [%d..%d]: ", l, r);
    for (int x = l; x <= r; x++) printf("%d ", arr[x]);
    printf("\\n");
}

void merge_sort(int arr[], int l, int r) {
    if (l >= r) return;
    int m = l + (r - l) / 2;
    depth++;
    merge_sort(arr, l, m);
    merge_sort(arr, m + 1, r);
    depth--;
    merge(arr, l, m, r);
}

int main() {
    int arr[] = {5, 2, 8, 1};
    printf("Sorting: 5 2 8 1\\n\\n");
    merge_sort(arr, 0, 3);
    return 0;
}
`} time="O(n log n)" space="O(n)" />

<Callout type="info">
Merge sort is **stable** (equal elements keep their relative order) and **guaranteed** O(n log n) regardless of input. The downside: it needs O(n) extra space for the temporary arrays.
</Callout>

## When to Use Merge Sort

| Pro | Con |
|-----|-----|
| Always O(n log n) | O(n) extra memory |
| Stable sort | Slower than quicksort in practice |
| Great for linked lists | More complex than basic sorts |
| Naturally parallelizable | |
