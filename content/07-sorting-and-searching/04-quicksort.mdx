Quicksort is typically the fastest comparison-based sorting algorithm in practice. It works by choosing a **pivot** element and partitioning the array around it.

## The Idea

1. **Pick a pivot** element
2. **Partition**: move all elements smaller than pivot to the left, larger to the right
3. **Recurse** on the left and right partitions

<Editor defaultCode={`#include <stdio.h>

void swap(int *a, int *b) {
    int temp = *a;
    *a = *b;
    *b = temp;
}

// Lomuto partition: pivot is last element
int partition(int arr[], int low, int high) {
    int pivot = arr[high];
    int i = low - 1;  // index of smaller element

    for (int j = low; j < high; j++) {
        if (arr[j] <= pivot) {
            i++;
            swap(&arr[i], &arr[j]);
        }
    }
    swap(&arr[i + 1], &arr[high]);
    return i + 1;  // pivot's final position
}

void quicksort(int arr[], int low, int high) {
    if (low < high) {
        int pi = partition(arr, low, high);
        quicksort(arr, low, pi - 1);
        quicksort(arr, pi + 1, high);
    }
}

int main() {
    int arr[] = {10, 7, 8, 9, 1, 5};
    int n = 6;

    quicksort(arr, 0, n - 1);

    for (int i = 0; i < n; i++) {
        printf("%d ", arr[i]);
    }
    printf("\\n");

    return 0;
}
`} time="O(n log n) average" space="O(log n) stack" />

<Complexity time="O(n log n) average, O(n²) worst" space="O(log n)" />

## How Partition Works

Let's trace through partitioning `[10, 7, 8, 9, 1, 5]` with pivot = 5:

```
i = -1, pivot = 5

j=0: arr[0]=10 > 5, skip
j=1: arr[1]=7 > 5, skip
j=2: arr[2]=8 > 5, skip
j=3: arr[3]=9 > 5, skip
j=4: arr[4]=1 <= 5, i=0, swap arr[0] and arr[4] → [1, 7, 8, 9, 10, 5]

Place pivot: swap arr[1] and arr[5] → [1, 5, 8, 9, 10, 7]
Pivot index: 1
```

Everything left of index 1 is ≤ 5, everything right is > 5.

## Hoare's Partition

An alternative partitioning scheme that's often faster (fewer swaps on average):

<Editor defaultCode={`#include <stdio.h>

void swap(int *a, int *b) {
    int t = *a; *a = *b; *b = t;
}

int hoare_partition(int arr[], int low, int high) {
    int pivot = arr[low];
    int i = low - 1;
    int j = high + 1;

    while (1) {
        do { i++; } while (arr[i] < pivot);
        do { j--; } while (arr[j] > pivot);
        if (i >= j) return j;
        swap(&arr[i], &arr[j]);
    }
}

void quicksort(int arr[], int low, int high) {
    if (low < high) {
        int p = hoare_partition(arr, low, high);
        quicksort(arr, low, p);       // note: include p
        quicksort(arr, p + 1, high);
    }
}

int main() {
    int arr[] = {10, 7, 8, 9, 1, 5};
    int n = 6;

    quicksort(arr, 0, n - 1);

    for (int i = 0; i < n; i++) {
        printf("%d ", arr[i]);
    }
    printf("\\n");

    return 0;
}
`} time="O(n log n) average" space="O(log n)" />

## Worst Case

<Callout type="warning">
Quicksort degrades to **O(n²)** when the pivot is always the smallest or largest element. This happens with sorted input and last-element pivoting. Mitigations:
- **Median-of-three**: pick the median of first, middle, last
- **Random pivot**: choose a random element
- **Introsort**: switch to heapsort if recursion depth exceeds O(log n)
</Callout>

## Quicksort vs Merge Sort

| | Quicksort | Merge Sort |
|---|-----------|------------|
| Average | O(n log n) | O(n log n) |
| Worst | O(n²) | O(n log n) |
| Space | O(log n) | O(n) |
| Stable | No | Yes |
| Cache | Excellent | Good |
| Practice | Usually fastest | Guaranteed performance |
