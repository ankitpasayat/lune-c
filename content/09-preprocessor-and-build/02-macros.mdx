Macros go beyond simple constants. Function-like macros perform text substitution with parameters — powerful but treacherous.

## Function-Like Macros

<Editor defaultCode={`#include <stdio.h>

#define SQUARE(x) ((x) * (x))
#define MAX(a, b) ((a) > (b) ? (a) : (b))
#define MIN(a, b) ((a) < (b) ? (a) : (b))
#define ABS(x) ((x) < 0 ? -(x) : (x))

int main() {
    printf("SQUARE(5) = %d\\n", SQUARE(5));
    printf("MAX(3, 7) = %d\\n", MAX(3, 7));
    printf("MIN(3, 7) = %d\\n", MIN(3, 7));
    printf("ABS(-42) = %d\\n", ABS(-42));

    return 0;
}
`} time="O(1)" space="O(1)" />

<Callout type="warning">
**Always parenthesize macro parameters and the whole expression!** Without parentheses, operator precedence can cause bugs. `SQUARE(x)` defined as `x * x` would make `SQUARE(1+2)` expand to `1+2 * 1+2 = 5` instead of `9`.
</Callout>

## Macro Pitfalls

### Double Evaluation

<Editor defaultCode={`#include <stdio.h>

#define SQUARE(x) ((x) * (x))

int main() {
    int a = 3;
    printf("SQUARE(a++) = %d\\n", SQUARE(a++));
    // Expands to: ((a++) * (a++))
    // a is incremented TWICE! Result is undefined behavior.

    printf("a is now %d (expected 4, got something else)\\n", a);

    return 0;
}
`} time="O(1)" space="O(1)" />

<Callout type="warning">
Never pass expressions with side effects (like `a++`) to macros. They get evaluated multiple times. Use inline functions instead when this matters.
</Callout>

### Missing Parentheses

```c
// BAD: missing outer parens
#define DOUBLE(x) (x) + (x)
int y = 10 * DOUBLE(3);  // 10 * (3) + (3) = 33, not 60!

// GOOD: fully parenthesized
#define DOUBLE(x) ((x) + (x))
int y = 10 * DOUBLE(3);  // 10 * ((3) + (3)) = 60 ✓
```

## Stringify and Token Pasting

Two special operators for macros:

<Editor defaultCode={`#include <stdio.h>

// # turns a parameter into a string literal
#define PRINT_VAR(x) printf(#x " = %d\\n", x)

// ## concatenates tokens
#define MAKE_FUNC(name) int get_##name() { return name; }

int width = 100;
int height = 200;
MAKE_FUNC(width)
MAKE_FUNC(height)

int main() {
    int a = 42;
    int b = 17;
    PRINT_VAR(a);  // expands to: printf("a" " = %d\n", a);
    PRINT_VAR(b);

    printf("width from func: %d\\n", get_width());
    printf("height from func: %d\\n", get_height());

    return 0;
}
`} time="O(1)" space="O(1)" />

## Multi-Line Macros

Use backslash for continuation and `do { ... } while(0)` for safety:

<Editor defaultCode={`#include <stdio.h>

// BAD: naked block can break in if-else
// #define LOG(msg) { printf("[LOG] "); printf(msg); printf("\\n"); }

// GOOD: do-while-0 idiom
#define LOG(msg) do { \\
    printf("[LOG] "); \\
    printf("%s", msg); \\
    printf("\\n"); \\
} while(0)

#define SWAP(a, b, type) do { \\
    type _temp = (a); \\
    (a) = (b); \\
    (b) = _temp; \\
} while(0)

int main() {
    LOG("Starting program");

    int x = 10, y = 20;
    printf("Before: x=%d, y=%d\\n", x, y);
    SWAP(x, y, int);
    printf("After:  x=%d, y=%d\\n", x, y);

    LOG("Done");
    return 0;
}
`} time="O(1)" space="O(1)" />

<Callout type="info">
The `do { ... } while(0)` wrapper ensures the macro works correctly inside `if` statements. Without it, `if (cond) LOG("x"); else ...` would break because the compiler sees the semicolons differently.
</Callout>

## Variadic Macros

Accept a variable number of arguments:

<Editor defaultCode={`#include <stdio.h>

#define DEBUG_PRINT(fmt, ...) \\
    fprintf(stderr, "[DEBUG %s:%d] " fmt "\\n", __FILE__, __LINE__, ##__VA_ARGS__)

int main() {
    int x = 42;
    float y = 3.14;

    DEBUG_PRINT("x = %d", x);
    DEBUG_PRINT("y = %.2f", y);
    DEBUG_PRINT("x = %d, y = %.2f", x, y);

    return 0;
}
`} time="O(1)" space="O(1)" />

## Macros vs Inline Functions

| Feature | Macros | Inline Functions |
|---------|--------|-----------------|
| Type safety | None | Full |
| Debug symbols | No | Yes |
| Evaluate args | Multiple times | Once |
| Code generation | Text replacement | Compiler decides |
| Scope | Global | Normal C scoping |

**Rule of thumb**: prefer `static inline` functions for most cases. Use macros only when you need text manipulation (`#`, `##`), conditional compilation, or generic-type behavior.
